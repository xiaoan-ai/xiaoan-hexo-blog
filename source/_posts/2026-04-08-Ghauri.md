# Ghauri

**一款高级跨平台工具，自动化检测与利用SQL注入安全漏洞的过程。**

![img](/images/notes/漏洞利用-ghauri/images/1.PNG)

## **系统需求**

- Python 3
- Python 的 `pip3`

## **安装**

- 进入 **ghauri** 目录。
- 安装依赖：`python3 -m pip install --upgrade -r requirements.txt`
- 安装程序：`python3 setup.py install` 或 `python3 -m pip install -e .`
- 通过简单的 `ghauri --help` 命令即可访问和运行 Ghauri。

## **下载 Ghauri**

您可以通过克隆GitHub仓库来获取Ghauri的最新版本。

```bash
git clone https://github.com/r0oth3x49/ghauri.git
```

## **功能特色**

- 支持多种注入载荷类型：

  - 基于布尔的注入
  - 错误基础注入
  - 时间基础注入
  - 堆叠查询

- 支持以下数据库管理系统（DBMS）的SQL注入：

  - MySQL
  - 微软SQL Server
  - PostgreSQL
  - Oracle
  - 微软Access（目前仅支持布尔型盲注的指纹识别）

- 支持以下类型的注入：

  - GET/POST请求注入
  - 标头注入
  - cookie注入
  - 多部分表单数据注入
  - JSON基础注入
  - SOAP/XML基础注入

- 支持代理选项 `--proxy`。

- 支持从txt文件解析请求：使用开关 `-r file.txt`。

- 支持限制提取数据库/表/列/转储的数据：开关 `--start 1 --stop 2`。

- 添加了各个阶段的恢复支持。

- 添加了跳过URL编码的开关：`--skip-urlencode`。

- 添加了在布尔/时间基于注入中验证提取字符的支持。

- 添加了根据用户需求处理重定向的支持。

- 添加了一个用于SQL交互命令行的实验性开关：`--sql-shell`。

- 添加了支持新鲜查询的开关：`--fresh-queries`。

- 添加了提取主机名的开关：`--hostname`。

- 添加了从GitHub更新Ghauri的开关：

  ```
  --update
  ```

  。

  - 注意：要使此开关适用于未来更新，Ghauri需从GitHub克隆/安装。 对于旧版本用户，如果使用Git安装，则需运行 `git pull` 获取更新； 之后的更新可通过运行 `ghauri --update` 命令获得Ghauri的最新版本。

## **高级使用**

```markdown
作者：Nasir Khan (<a href="https://pk.linkedin.com/in/r0oth3x49">r0ot h3x49</a>)

用法：ghauri -u URL [OPTIONS]

这是一个基于Python的跨平台高级SQL注入检测与利用工具。

一般选项：
  -h, --help 显示帮助信息。
  --version 显示版本信息。
  -v VERBOSE 日志详细程度：1-5（默认1）。
  --update 更新Ghauri。
  --batch 不询问用户输入，采用默认行为。
  --flush-session 清除当前目标的会话文件。
  --fresh-queries 忽略会话文件中存储的查询结果。
  --test-filter 通过标题选择测试载荷（实验性）。

目标定义：
  需至少提供以下选项之一以定义目标：

  -u URL, --url URL 目标URL（如'http://www.example.com/vuln.php?id=1'）。
  -r REQUESTFILE 从文件加载HTTP请求。

连接选项：
  这些选项可用于指定如何连接到目标URL。

  -A, --user-agent HTTP用户代理头部值。
  -H, --header 额外的头部（例如"X-Forwarded-For: 127.0.0.1"）。
  --host HTTP Host头部值。
  --data POST方式发送的数据字符串（如"id=1"）。
  --cookie HTTP Cookie头部值（如"PHPSESSID=a8d127e.."）。
  --referer HTTP 引用页头部值。
  --headers 额外的头部信息（如"Accept-Language: fr\nETag: 123"）。
  --proxy 使用代理连接至目标URL。
  --delay 每个HTTP请求之间的延迟秒数。
  --timeout 超时前等待的秒数（默认30秒）。
  --retries 发生连接错误时的重试次数（默认3次）。
  --confirm 确认注入的有效负载。
  --skip-urlencode 跳过载荷数据的URL编码。
  --force-ssl 强制使用SSL/HTTPS。

性能优化：
  这些选项可用来优化Ghauri的性能。

  --threads THREADS 最大并发HTTP(s)请求数量（默认1）。

注入参数：
  这些选项用于指定要测试的参数，提供自定义注入载荷及篡改脚本。

  -p TESTPARAMETER 可测试的参数。
  --dbms DBMS 强制后端DBMS为给定值。
  --prefix 注入载荷前缀字符串。
  --suffix 注入载荷后缀字符串。
  --safe-chars 跳过特定字符的URL编码（例如：--safe-chars="[]"）。
  --fetch-using 使用不同的操作符获取数据（例如：--fetch-using=between/in）。

检测定制：
  这些选项用于定制检测阶段。

  --level LEVEL 执行测试的级别（1-3，默认1）。
  --code CODE 当查询评估为真时匹配的HTTP状态码。
  --string 当查询评估为真时匹配的字符串。
  --not-string 当查询评估为假时匹配的字符串。
  --text-only 仅基于文本内容比较页面。

技术调整：
  这些选项用于微调特定SQL注入技术的测试。

  --technique TECH 使用的SQL注入技术（默认"BEST"）。
  --time-sec TIMESEC 数据库响应延迟的秒数（默认5）。

枚举：
  这些选项用于枚举后端DBMS的信息、结构及其表中的数据。

  -b, --banner 获取DBMS标语。
  --current-user 获取当前DBMS用户。
  --current-db 获取当前DBMS数据库。
  --hostname 获取DBMS服务器的主机名。
  --dbs 枚举DBMS数据库。
  --tables 枚举DBMS数据库表。
  --columns 枚举DBMS数据库表的列。
  --dump 转储DBMS数据库表条目。
  -D DB 要枚举的DBMS数据库。
  -T TBL 要枚举的DBMS数据库表（复数）。
  -C COLS 要枚举的DBMS数据库表列（复数）。
  --start 为dbs/tables/columns/dump检索条目的起始位置。
  --stop 为dbs/tables/columns/dump检索条目的结束位置。
  --sql-shell 提供交互式SQL外壳（实验性）。

示例：
  ghauri -u http://www.example.com/vuln.php?id=1 --dbs
```
